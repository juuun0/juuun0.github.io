## Instruction Set Architecture

Instruction Set Architecture(**ISA**) is an abstract model of computer that how device is execute instruction likes. Many of enduser desktop is using **intel** or **amd** cpu and is based on **CISC** isa by the way most of mobile device is **arm** cpu and it is based on **RISC**.

## CISC

**Complex Instruction Set Computer** is more famillar to human perspective better than machine perspective in my think. If you learning assembly as firstly to linux or windows binary exploitation you may met CISC type assembly becaus of intel or amd is based on this.

Computer machine prefer to more efficient processing but CISC instruction has variable length that's the reason why i think it's better famillar to human perspective. Due to these features CISC is slower than RISC cpu but has more compatieability.

## RISC

**Reduced Instrucion Set Computer** is mostly used in mobile, embeded device cpu because of lower power required to maintain cpu. This is using fixed length instruction so it makes more faster. If you trying to IoT or mobile devices binary analysis, you need to know RISC type assembly.
